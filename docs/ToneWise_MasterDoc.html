Problem Statement

Every day, millions of cross-cultural knowledge workers — engineers, analysts, PMs — write dozens of messages in Slack, Teams, Gmail, and Jira. Many of them open a separate AI assistant, paste their draft, ask it to polish the tone, then copy it back. This context-switching is expensive, repetitive, and entirely unsolved by a single purpose-built product.

Real User Scenario

An Indian engineer working at an American firm writes: "i already told you guys this is a bug it needs to be fixed today" — and before hitting send, opens Copilot to rewrite it as professional, firm-but-respectful English. This happens 10–30 times a day per user. There is no dedicated, zero-friction tool for this.

The gap this product fills

Current Behaviour Pain Point ToneWise Solves
Open ChatGPT / Copilot tab Context switch breaks flow Inline in the same window
Write a long prompt every time No memory of style preferences One-click tone presets
General LLM makes it too formal Loses the original point Intent-preserving rewrite
No grammar check in context Embarrassing typos sent Grammar layer built in
Requires login / subscription Barrier to adoption 10 free daily uses, no login
02

Product Vision

ToneWise is a zero-login, IP-rate-limited web application and browser extension that rewrites workplace messages — Slack, Teams, Gmail, Jira comments — with the correct tone for the context. It operates as a lightweight AI layer on top of free-tier LLMs, delivering 10 rewrites per user per 24-hour window with no account required.

North Star Statement

Every professional should be able to communicate confidently across cultural and linguistic barriers without switching contexts, installing enterprise software, or paying a subscription — starting on day one.

Hooks, Agents & Skills

This section defines every Claude Code construct that will be built for ToneWise. Claude Code will generate, test, and wire these automatically based on this specification.

Agents

Main Agent
ToneOrchestrator

Receives the raw message + preset. Routes to classifier, then rewrite agent. Enforces quality gates before returning output.

Sub-Agent
IntentClassifier

Reads the user's original message and identifies: is the user venting, asking, escalating, or explaining? Determines clarification need.

Sub-Agent
ClarificationAgent

Activated only in Mode B. Generates 1–2 targeted questions to resolve ambiguity. Strict output format — never more than 2 questions.

Sub-Agent
RewriteAgent

Produces the final rewritten message. Takes: original text, tone preset, platform context, intent classification, and any clarification answers.

Sub-Agent
GrammarAgent

Post-processing pass. Fixes spelling, grammar, punctuation. Does not change tone or meaning. Runs on every output regardless of mode.

Hooks

Pre-Processing Hook
InputSanitizer

Strips PII patterns (email addresses, phone numbers, names) before sending to LLM. Adds them back to output post-rewrite using placeholder tokens.

Pre-Processing Hook
LanguageDetector

Detects input language. If non-English, translates to English first, processes, then optionally returns in original language.

Post-Processing Hook
LengthGuardrail

Ensures output is never longer than 1.5× the input (for chat modes). Triggers a compression pass if exceeded. Chat messages must stay concise.

Post-Processing Hook
ToneVerifier

Scores output on requested tone dimensions (politeness, assertiveness) using a lightweight classifier. Flags if output drifts from preset.

Infrastructure Hook
IPRateLimiter

Checks Redis for IP request count in rolling 24h window. Returns 429 with time-until-reset if limit reached. Logs request fingerprint (IP + UA hash).

Infrastructure Hook
UsageLogger

Non-blocking async logger. Records: hashed IP, timestamp, preset used, input length, output length. Never stores message content.

Skills

Skill
ToneMapping

Maps each preset to a structured system prompt with specific do/don't rules, example rewrite pairs, and assertiveness calibration instructions.

Skill
PlatformAwareness

Injects platform-specific formatting rules: Slack (no salutation, ≤3 sentences), email (full structure), Jira (no emotion, action verbs).

Skill
CulturalCalibration

Library of US/UK/APAC corporate communication norms. Used to ensure output sounds native to the target business culture, not just translated.

Skill
FewShotExamples

Dynamic retrieval of 2–3 in-context examples matching the user's platform + tone combination. Improves rewrite quality with minimal tokens.

Rate Limiting & IP Strategy

Design Principle

Users get 10 rewrites per 24-hour rolling window with no account creation. The system must be abuse-resistant without being hostile to legitimate users. No dark patterns.

Implementation Plan

// Edge function: /api/rewrite (Next.js App Router) import { Ratelimit } from "@upstash/ratelimit"; import { Redis } from "@upstash/redis"; const ratelimit = new Ratelimit({ redis: Redis.fromEnv(), limiter: Ratelimit.slidingWindow(10, "24 h"), analytics: true, prefix: "tonewise" }); export async function POST(req) { const ip = req.headers.get("x-forwarded-for") ?? "anonymous"; const { success, reset, remaining } = await ratelimit.limit(ip); if (!success) { return Response.json({ error: "Daily limit reached", resetsAt: new Date(reset).toISOString(), upgradeUrl: "/upgrade" }, { status: 429 }); } // proceed to agent pipeline... }

Fingerprinting Strategy

IP alone is insufficient (shared NAT, VPNs). The system combines IP + browser fingerprint hash (User-Agent, Accept-Language headers, screen resolution via JS) to create a composite identifier. This is stored as a one-way hash — not reversible to any personal data.

SignalWeightWhy
IPv4/IPv6PrimaryMost reliable, no JS needed
User-Agent hashSecondaryDifferentiates devices on same IP
Accept-LanguageSupplementaryLocale signal, low collision risk
Canvas fingerprintV2 onlyHigh accuracy, requires user notice
Privacy Note

No fingerprint data is linked to identity. All identifiers expire with the 24-hour window. GDPR/PDPA compliance is maintained by storing only hashed composites, never raw IPs in long-term storage.

LLM Safety Guardrails

Adversarial Input Handling

The InputSanitizer hook strips common prompt injection patterns before the message reaches the LLM. The system prompt explicitly instructs the agent to ignore instructions embedded in user messages and only perform the rewrite task. Claude Haiku's built-in safety filters provide the second line of defence.
