Agent Contracts (TypeScript)

packages/agents/src/types/index.ts typescript
export type TonePreset = | 'humble-polite' | 'confident-direct' | 'clarifying' | 'empathetic-nudge' | 'grammar-only'; export type Platform = | 'slack' | 'email' | 'jira' | 'linkedin'; export type Intent = | 'asking' | 'escalating' | 'explaining' | 'venting' | 'neutral'; export interface RewriteContext { original: string; preset: TonePreset; platform: Platform; intent?: Intent; needsClarification?: boolean; clarificationAnswers?: string[]; sanitized?: string; // PII stripped version rewritten?: string; // output from RewriteAgent final?: string; // after grammar pass } export interface AgentResult { success: boolean; context: RewriteContext; error?: string; }
Agent 1
IntentClassifier
Reads the raw message. Returns one of 5 intent labels. Sets needsClarification=true if message is ambiguous and Mode B is requested.
in: original, preset → out: intent, needsClarification
Agent 2
RewriteAgent
Core rewrite. Builds system prompt from toneMapping skill + platformAwareness skill + 2–3 fewShot examples. Calls Claude Haiku with streaming.
in: sanitized, preset, platform, intent → out: rewritten
Agent 3
GrammarAgent
Post-processing pass. Fixes spelling and punctuation. Strict instruction: do not change tone, meaning, or length. Runs on every output.
in: rewritten → out: final
Agent 2b (Mode B)
ClarificationAgent
Generates exactly 1–2 questions when intent is unclear. Strict output format enforced via structured prompt. Never asks more than 2.
in: original, intent → out: questions[]
04

Hooks Architecture

Hooks are pure functions that run before or after agents. They never call the LLM. Pre-hooks can mutate the context or abort the pipeline. Post-hooks run after the final output is produced.

HookStageCan abort?What it does
IPRateLimiterPre✓ → 429Checks Upstash Redis for IP request count in 24h window
InputSanitizerPre✗Strips email/phone/name patterns, replaces with [REDACTED_n]
LanguageDetectorPre✗Detects non-English input; translates before processing
LengthGuardrailPost✗If output > 1.5× input, triggers compression re-call
ToneVerifierPost✗Scores output vs preset. Flags drift (logged, not blocking)
UsageLoggerPost✗Non-blocking async: logs hashed IP, timestamp, preset, lengths
Important — PII Handling

InputSanitizer replaces PII with tokens like [REDACTED_1] before any LLM call. After the final output is produced, it runs a restore pass replacing tokens back with original values. Message content is never stored at any point.

Data Flow & State

ToneWise is stateless by design. No session, no database, no auth. The only persistent state is in Upstash Redis (rate limit counters) and PostHog (aggregate analytics). Everything else is ephemeral per-request.

07

Browser Extension Architecture

Built with Plasmo framework. Single codebase compiles to Chrome MV3 and Firefox MV2. The extension injects a floating button into detected text fields on supported platforms.

apps/extension/
apps/extension/
├── popup.tsx— extension popup UI (same components as web)
├── contents/
│ └── tonewise-injector.tsx— content script, detects text fields
├── background.ts— service worker (MV3)
├── package.json
└── plasmo.config.ts
Content Script Logic

The injector listens for focusin events on textarea and [contenteditable] elements. On Slack/Gmail/Jira (matched by hostname), it injects a floating ToneWise button near the active field. Clicking the button opens a mini popup inline — no redirect to the web app needed.

POSTHOG_KEY phc_xxx... posthog.com → Project → API Key (optional)
Never commit .env.local

The .env.local file is in .gitignore. Only .env.example is committed — it contains placeholder values and comments but no real secrets.

Local Dev Setup

Full setup from a fresh clone to running UI in browser — estimated time: 8 minutes.

Terminal — one-time setup bash
# 1. Clone and install git clone https://github.com/YOUR_ORG/tonewise.git cd tonewise pnpm install # installs all workspaces # 2. Set up environment cp apps/web/.env.example apps/web/.env.local # → open .env.local and fill in your keys (see Section 08) # 3. Verify everything builds pnpm typecheck # should show 0 errors pnpm lint # should show 0 warnings pnpm test # unit tests — should all pass # 4. Run the web app cd apps/web pnpm dev # starts on http://localhost:3000
Terminal — E2E tests (separate tab) bash
# Run this once to install browser binaries npx playwright install # Then run E2E tests (app must be running on :3000) pnpm test:e2e # Or run with UI mode for debugging pnpm test:e2e --ui
